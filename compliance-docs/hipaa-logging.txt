HIPAA Security Rule — Audit Controls (Logging)
45 CFR §164.312(b)

Standard: Audit Controls
A covered entity or business associate must implement hardware, software, and/or
procedural mechanisms that record and examine activity in information systems that
contain or use electronic protected health information (ePHI).

Implementation Guidance:
The audit controls standard requires covered entities to implement mechanisms to
record and examine access and other activity in information systems that contain or
use ePHI. The entity must:

1. Determine the activities that will be tracked or audited — including read access,
   write access, deletions, and administrative actions on ePHI.
2. Select and implement the tools and mechanisms to record and monitor these activities.
3. Develop and deploy policies and procedures to regularly review audit logs.
4. Retain audit logs for a minimum of 6 years as required by §164.530(j).

For AWS workloads containing ePHI:
- S3 server access logging must be enabled on all buckets storing ePHI
- CloudTrail must be enabled to capture API-level activity
- RDS audit logging and enhanced monitoring should be enabled
- VPC Flow Logs must be enabled for network-level audit trails
- All logs must be stored in a tamper-proof location (separate S3 bucket with
  object lock or write-once policies)
- Access logs serve as the primary evidence during HIPAA compliance audits and
  OCR investigations
